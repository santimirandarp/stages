A connected World

Succesive approximations: a simple model

Let's list the necessary components:

  • Huge Network (the cloud, or the World Wide Web)
  • A Client (a program in your computer)
  • A Server (a program in other computer)

This is indeed the sort of picture I have in mind:

A step further

I like to think of the Cloud as the countries, cities, towns, houses; they themselves connected through highways, roads, streets. (Or did Elon Musk invented something new?!). The network/cloud allows communication, but in this case, between computers.

We could conversely think of the "physical" world of cities, and so on, as a network. A network is just a bunch of connected parts, called Nodes. Each node would be, for example, a city.

This communication, as we listed, needs not only the Cloud, but a server and a client. Again, if we think of The Cloud as streets and houses, then the server is the market (provides a service), and you (your browser) is the client.

So the server and the client are 2 computers communicated over the cloud. This is the most crazy idea here.

I will just turn it slightly crazier. Web servers (the market), and Web Browsers (you) are not the only type of connection between 2 computers. But this is a topic for another post.

Wrapping up

Once we start a server on a computer, it can connect to other computers. A web server is basically a server that can be reached using a Web Browser.

(Because a server can connect to other computers, a server can be hacked out. But it's not that easy.)

The most interesting part is that we can share information between 2 different computers (this is phones, PCs, etc.)

The page you're seeing right now is "hosted" in a server. It means that this is much like seeing a Word Document, but the file is actually in my -as opposed to your- computer.
